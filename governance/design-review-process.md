# Design Review Process

If your request has been stopped by the brand enforcement system, or if you want to propose a new design pattern, color, component, or asset type, use this process.

---

## When You Need a Review

Submit a design review request if you want to:

- Create assets that aren't covered by an existing template
- Use a color, font, icon, or layout not in the current design system
- Build public-facing materials from scratch
- Propose a change or addition to the brand system
- Request a new template for a new channel or format
- Use a partner or co-brand lockup
- Create anything that triggered a STOP message

---

## How to Submit

### Via Email
**To:** Marketing@demandscience.com
**Subject:** Design Review Request — [Brief description]

Include in your email:
1. **What you need** — describe the asset and its purpose
2. **Who it's for** — internal team, client, prospect, public
3. **When you need it** — hard deadline if applicable
4. **Why existing templates don't work** — what gap you're trying to fill
5. **Reference examples** — links or screenshots if helpful

### Via Slack
Post in **#design-team** with the same information above.

---

## What to Expect

| Timeline | What happens |
|----------|-------------|
| Within 1 business day | Design acknowledges the request |
| Within 3 business days | Design provides a decision or asks for more information |
| Complex requests | May require additional stakeholder review — Design will set a timeline |

---

## Possible Outcomes

| Decision | What it means |
|----------|--------------|
| ✅ **Approved** | Proceed as requested. Design may provide specific direction. |
| ✅ **Approved with modifications** | Proceed using the changes Design specifies. |
| ❌ **Rejected** | Do not proceed. Design will provide an alternative solution. |
| 🔄 **Escalated** | Requires senior review. Design will manage and communicate the outcome. |
| 💬 **More info needed** | Design needs clarification before deciding. Respond promptly to unblock. |

---

## Proposing a System Change

If you want to propose a new color, component, layout, or rule for the design system:

1. Document your proposal clearly — what, why, and what problem it solves
2. Submit via email or Slack with **[System Change Proposal]** in the subject
3. Design will review against brand strategy and existing system
4. If approved, Design will update the relevant files and log it in `CHANGELOG.md`
5. All system changes are versioned — nothing changes without a changelog entry
