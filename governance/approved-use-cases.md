# Approved Use Cases

A quick reference for what requires design review and what doesn't.

---

## ✅ Approved — No Review Required

These can be built immediately using the existing system:

| Asset type | Condition |
|-----------|-----------|
| Internal presentation | Uses the approved slide template · White backgrounds · Correct fonts and colors |
| Client-facing deck | Uses the approved slide template · No new layouts or components |
| One-pager / brief | Uses approved doc template · Correct margins, fonts, colors |
| Internal report | Standard layout · Approved type and color |
| Chart / data visual | Approved color order · White background · Correct fonts |
| Icon usage | Google Material Symbols Rounded · Navy or Electric Blue · Correct size |
| Email (using template) | Follows approved email structure |
| Social post (using template) | Follows approved social dimensions and brand |
| Internal Slack/comms graphic | Standard brand colors and fonts · Not for external use |
| Web page (using component library) | Uses approved components, tokens, and SEO standards |

---

## 🔴 Always Requires Design Approval

| Asset type | Why |
|-----------|-----|
| New logo treatment or variant | Logo integrity is non-negotiable |
| Co-brand / partner lockup | Brand combination rules must be reviewed |
| New color or font | Must be evaluated against the full system |
| New icon from non-approved library | Library lock is absolute |
| New slide or doc template | Must be validated for brand consistency before distribution |
| Event or campaign visual identity | Needs Design direction from the start |
| Public infographic (new layout) | High brand visibility — must be reviewed |
| Video lower thirds / motion graphics | Motion and brand standards must be applied |
| Any asset with "this is just a quick one-off" in the description | One-offs become standards — all external assets need review |
| Any asset for a new channel not in the design system | Channel guide must be created or updated first |
